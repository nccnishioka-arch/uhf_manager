# Contributing Guide

## 1. Version Policy

Major.Minor.Patch

- **Major**
  - Product release
  - Large architecture changes

- **Minor**
  - New features
  - Refactoring

- **Patch**
  - Bug fixes
  - Small improvements

### Example Roadmap

| Version | Contents |
|---------|----------|
| 0.12.x  | Refactoring |
| 0.13.x  | Core Architecture, Data Model |
| 0.14.x  | Business Features |
| 1.0.0   | Production Release |

## 2. Branch Naming

Use the following naming convention.

| Prefix | Usage |
|--------|-------|
| `feature/` | New features |
| `fix/` | Bug fixes |
| `refactor/` | Refactoring |
| `docs/` | Documentation updates |
| `release/vX.XX.X` | Release preparation |
| `hotfix/vX.XX.X` | Hotfixes |

Examples:

- `feature/add-reader-communication`
- `fix/config-load-error`
- `refactor/data-model`
- `docs/update-contributing`
- `release/v0.14.0`
- `hotfix/v0.12.7`
